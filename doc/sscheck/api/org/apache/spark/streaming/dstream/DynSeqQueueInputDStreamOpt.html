DynSeqQueueInputDStreamOpt

class DynSeqQueueInputDStreamOpt[A] extends InputDStream[Option[A]]

Alternative version of DynSeqQueueInputDStreamOpt that always emits some data. To do that a singleton batch with the record None is emitted for empty batches, and batches with Some are used otherwise. Note this implies that an absent batch cannot be distinguish from an empty batch.

Currently this class is not used in DStreamProp, if it is needed in the future we could create a new sealed trait with 3 case classes to distinguish between no batch generated, empty batch and batch with records. That might make more sense for test case multiplexion
